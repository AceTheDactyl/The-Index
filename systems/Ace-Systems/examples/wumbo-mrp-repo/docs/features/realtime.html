Optimization Strategies

Phase 1: Current Implementation DONE

  • TypedArray operations for pixel data
  • Minimal DOM access during processing
  • Pre-computed lookup tables for APL encoding
  • Inline physics calculations

Phase 2: Web Workers PLANNED

  • Offload encoding to dedicated worker thread
  • Non-blocking main thread during processing
  • Parallel decoding of redundant payloads
  • Message passing optimization

Phase 3: WebGL Acceleration PLANNED

  • GPU-based LSB embedding for large images
  • Shader-based physics computation
  • Parallel pixel processing
  • 10-100× speedup for 1024×1024+ images

Streaming Encode

// Encode patterns as stream
async function* encodeStream(zValues) {
    for (const z of zValues) {
        const pattern = WumboMRP.createPattern({ z });
        const carrier = WumboMRP.generateEncodedCarrier({ z });
        yield carrier.dataUrl;
        await new Promise(r => setTimeout(r, 16));  // Yield to main thread
    }
}

Batch Processing

// Process multiple patterns in parallel (Phase 2)
async function batchEncode(patterns) {
    const workers = navigator.hardwareConcurrency || 4;
    const chunks = splitArray(patterns, workers);

    const results = await Promise.all(
        chunks.map(chunk =>
            workerPool.run('encode', chunk)
        )
    );

    return results.flat();
}
