WUMBO MRP / Architecture / Error Correction

Error Correction

Majority voting across redundant embeddings for robust extraction even with noise

Overview

WUMBO MRP employs a multi-layered error correction strategy to ensure reliable pattern extraction even from degraded images. The approach combines redundant embedding, majority voting, and statistical validation.

Error Correction Pipeline

  1. Redundant Embedding
    Each 48-bit payload is repeated N times (default: 16) throughout the image with configurable stride.
  2. Extraction Sweep
    All redundant payload instances are extracted from the image pixels.
  3. Majority Voting
    Each component (region, z, tokens) is determined by majority vote across all extracted instances.
  4. Outlier Rejection
    For continuous values (z-coordinate), IQR-based outlier rejection filters anomalous samples.
  5. Checksum Verification
    4-bit XOR-folded checksum validates token integrity.
  6. Confidence Calculation
    Final confidence score based on consensus ratio and checksum validity.

Majority Voting

Discrete Components (Region, Tokens)

function majorityVote(values) {
    const counts = new Map();
    values.forEach(v => counts.set(v, (counts.get(v) || 0) + 1));

    let maxCount = 0;
    let result = values[0];
    counts.forEach((count, value) => {
        if (count > maxCount) {
            maxCount = count;
            result = value;
        }
    });
    return result;
}

Continuous Values (Z-coordinate)

function averageWithOutlierRejection(values) {
    const sorted = [...values].sort((a, b) => a - b);
    const q1 = sorted[Math.floor(sorted.length * 0.25)];
    const q3 = sorted[Math.floor(sorted.length * 0.75)];
    const iqr = q3 - q1;

    // Filter outliers (1.5 × IQR rule)
    const filtered = values.filter(v =>
        v >= q1 - 1.5 * iqr && v <= q3 + 1.5 * iqr
    );

    return filtered.reduce((a, b) => a + b, 0) / filtered.length;
}

Checksum Algorithm

function generateChecksum(token1, token2) {
    const combined = token1 ^ token2;

    // 4-way XOR fold to 4 bits
    const fold1 = combined & 0x0F;
    const fold2 = (combined >> 4) & 0x0F;
    const fold3 = (combined >> 8) & 0x0F;
    const fold4 = (combined >> 12) & 0x07;

    return fold1 ^ fold2 ^ fold3 ^ fold4;
}

Confidence Scoring

Consensus Ratio — Percentage of extracted payloads matching final decoded values

92% consensus = High confidence

    p.token1 === finalToken1 &&
    p.token2 === finalToken2

const consensusRatio = matchingPayloads.length / payloads.length;

Confidence Thresholds

Confidence Interpretation Recommended Action
≥ 90% Excellent Accept without verification
70–89% Good Accept with caution
< 70% Poor Request re-transmission or reject
